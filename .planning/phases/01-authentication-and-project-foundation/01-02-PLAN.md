  <name>Task 1: Auth modules -- Duo MFA wait, session detection, and SSO login flows</name>
  <files>src/auth/duo-wait.ts, src/auth/session.ts, src/auth/login.ts</files>
  <action>
1. Create src/auth/duo-wait.ts -- Duo MFA polling loop:
   - Export `waitForDuoApproval(page: Page, successUrlPattern: string, timeoutMs?: number): Promise<boolean>`
   - Default timeout: 15_000ms (per user decision: 15 seconds)
   - Uses `page.waitForURL(successUrlPattern, { timeout: timeoutMs })` -- NOT waitForNavigation (deprecated), NOT page.pause() (opens Inspector)
   - Returns true if URL matches within timeout, false on timeout
   - Import log from "../utils/log" for status messages

2. Create src/auth/session.ts -- session validity detection:
   - Export `isSessionValid(page: Page, targetUrl: string): Promise<boolean>`
   - Implementation: `page.goto(targetUrl, { waitUntil: "domcontentloaded", timeout: 10_000 })`
   - Check current URL after navigation: if URL contains "shibboleth", "login", or "idp" then session is NOT valid (redirected to login page)
   - Return true if still on target domain (not redirected to login)
   - Wrap in try/catch -- return false on any error (network failure, timeout)
   - Import log from "../utils/log"

3. Create src/auth/login.ts -- SSO login flows for both systems:
   - Import: Page from "playwright", log from "../utils/log", waitForDuoApproval from "./duo-wait", validateEnv from "../utils/env"

   Export `loginToUCPath(page: Page): Promise<boolean>`:
   - log.step("Navigating to UCPath...")
   - Navigate to https://ucpath.ucsd.edu
   - log.step("Login page loaded")
   - Click the "Log in to UCPath" link/button -- use `page.getByRole('link', { name: /log in/i })` or `page.getByText('Log in to UCPath')`. Do NOT use exact element IDs (PeopleSoft generates dynamic IDs per RESEARCH.md pitfall 1).
   - If a campus/institution selector appears, select "UC San Diego" -- use `page.getByText('UC San Diego')` or similar. PeopleSoft may have a dropdown or link.
   - Wait for SSO login page to load (URL should contain "shibboleth" or "login.ucsd.edu")
   - Get credentials from validateEnv()
   - log.step("Entering credentials...")
   - Fill username field: `page.getByLabel('Username')` or `page.locator('input[name="j_username"]')` -- SSO pages typically use this name
   - Fill password field: `page.getByLabel('Password')` or `page.locator('input[name="j_password"]')`
   - Click submit/login button: `page.getByRole('button', { name: /log in|sign in|submit/i })`
   - log.waiting("Waiting for Duo approval...")
   - Call waitForDuoApproval(page, "**/ucpath.ucsd.edu/**", 15_000)
   - If first attempt fails: log.waiting("Retrying -- waiting for Duo approval..."), retry once with same timeout
   - If second attempt fails: log.error("Duo approval timed out after two attempts"), return false
   - log.success("UCPath authenticated")
   - Return true

   Export `loginToACTCrm(page: Page): Promise<boolean>`:
   - log.step("Navigating to ACT CRM...")
   - Navigate to https://act-crm.my.site.com (the onboarding portal)
   - Check if already on an authenticated page (not redirected to login): if page URL does not contain "login", return true immediately with log.success("ACT CRM already authenticated (SSO session)")
   - If login page detected:
     - log.step("ACT CRM requires separate login")
     - Look for "Active Directory" option in login dropdown -- per user decision, use `page.getByText('Active Directory')` or locate a dropdown/select and choose that option. Salesforce Experience Cloud may use a custom component, so also try `page.locator('select').selectOption({ label: 'Active Directory' })` or clicking a visible option.
     - Get credentials from validateEnv()
     - log.step("Entering credentials...")
     - Fill username and password fields (Salesforce login forms typically use `input[name="username"]` and `input[name="password"]`)
     - Click login button
     - log.waiting("Waiting for Duo approval...")
     - Call waitForDuoApproval with appropriate URL pattern for ACT CRM domain
     - If timeout after retry: log.error("ACT CRM Duo approval timed out"), return false
   - log.success("ACT CRM authenticated")
   - Return true

   IMPORTANT NOTES on selectors (per RESEARCH.md Open Questions):
   - All selectors above are BEST-GUESS based on standard SSO/PeopleSoft/Salesforce patterns
   - PeopleSoft generates dynamic IDs -- never use exact IDs like #ICMyID_123
   - Prefer text-based (getByText, getByRole) and attribute-partial ([id*="KEYWORD"]) selectors
   - These WILL need adjustment during the first live test run. Wrap locators so they are easy to find and update.
   - Add comments like `// SELECTOR: may need adjustment after live testing` near each locator

Create src/cli.ts -- the CLI entry point that orchestrates the full auth flow:

1. Import commander's Command class
2. Import: launchBrowser, saveSession, clearSession from "./browser/launch"
3. Import: validateEnv from "./utils/env"
4. Import: log from "./utils/log"
5. Import: isSessionValid from "./auth/session"
6. Import: loginToUCPath, loginToACTCrm from "./auth/login"
7. Import: LoginOptions, AuthResult from "./auth/types"

8. Set up CLI program:
   ```
   program.name("hr-auto").description("UCPath HR Automation Tool").version("0.1.0")
   ```

9. Register `test-login` command:
   - Description: "Test authentication to UCPath and ACT CRM"
   - Option: `--fresh` (boolean, "Force fresh login, ignore saved session")

10. test-login action handler (async):
    a. Validate env FIRST (fail early per user decision):
       - Call validateEnv() -- exits if missing vars

    b. Launch browser:
       - If --fresh flag: call clearSession() first, then launchBrowser(true)
       - Otherwise: launchBrowser(false) (loads saved state if available)

    c. Check UCPath session:
       - Call isSessionValid(page, "https://ucpath.ucsd.edu")
       - If valid: log.success("UCPath session valid -- skipping login")
       - If NOT valid (or --fresh):
         - On stale session: call clearSession(), close context, relaunch browser without state (per user decision: silently clear and fall back to full login)
         - Call loginToUCPath(page)
         - If login fails: log.error("UCPath authentication failed"), close browser, process.exit(1)

    d. Check ACT CRM session:
       - Call isSessionValid(page, "https://act-crm.my.site.com")
       - If valid: log.success("ACT CRM session valid -- skipping login")
       - If NOT valid:
         - Call loginToACTCrm(page)
         - If login fails: log.error("ACT CRM authentication failed"), close browser, process.exit(1)

    e. Save session:
       - Call saveSession(context)
       - log.success("Session saved to .auth/")

    f. Print summary:
       - log.success("Authentication complete")
       - log.step("UCPath: authenticated")
       - log.step("ACT CRM: authenticated")

    g. Close browser (per user decision: close after successful test-login):
       - await browser.close()

    h. Error handling -- wrap entire flow in try/catch:
       - On browser crash / page load failure: retry ONCE (per user decision). Close browser, relaunch, retry the full flow. If second attempt fails, exit with error.
       - On any unexpected error: log.error with message, close browser, process.exit(1)

11. Call program.parse() at bottom of file
