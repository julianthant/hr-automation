## Summary

Added CRM record page field extraction and integrated onboarding tracker into CLI workflow. Live-verified CRM selectors and person search logic against production systems.

## What was built

- `extractRecordPageFields` function in `src/workflows/onboarding/extract.ts` — extracts department number and recruitment number from CRM ViewOnboarding page using multi-label fallback
- CLI `start-onboarding` command now calls `extractRecordPageFields` after `selectLatestResult` and before `navigateToSection("UCPath Entry Sheet")`
- Tracker row written to `./onboarding-tracker.xlsx` with masked SSN, rehire status, and workflow progress
- Person search logic fixed: determination based on post-Search dialog (dialog = new hire) vs results table (rehire)
- Duo MFA polling improved: 15s intervals instead of 60s for faster "Yes, this is my device" detection
- Screenshots added at each person search step for debugging

## Deviations

- Person search logic required multiple iterations to get right. Final logic: always click magnifying glass (dismiss dialog if present), always click Search, then check post-Search result — dialog = new hire, results table = rehire
- Duo "Yes, this is my device" was taking 60s to detect; changed to 15s polling intervals
