## Decisions Made
- **ExcelJS column key re-mapping:** ExcelJS loses column `key` properties after `readFile()`. When appending rows via `addRow(object)`, the object keys cannot be mapped to columns, resulting in silent data loss. Fix: re-apply `col.key` from TRACKER_COLUMNS after reading an existing worksheet.
- **Last-match regex for department number:** Used `matchAll` with last-match selection to handle edge cases where department text contains multiple parenthesized segments (e.g., "Some (text) Dept (000412)").

- **Found during:** Task 1 (GREEN phase, updateTracker append test)
- **Issue:** ExcelJS `getWorksheet()` after `readFile()` returns a sheet with empty column keys. `addRow({ firstName: "Bob", ... })` silently fails because the key-to-column mapping is absent.
- **Fix:** Added else-branch in `updateTracker` that re-applies `col.key` from TRACKER_COLUMNS for each column when reading an existing worksheet.
