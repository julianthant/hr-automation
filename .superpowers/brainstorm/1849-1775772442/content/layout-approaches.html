A: Enhanced Table (evolve current)

Keep the table layout, fix it with shadcn. Click rows to expand inline logs. Stats bar on top, filters below header. Familiar pattern, minimal learning curve.

Pros

  • Familiar — similar to current
  • Dense data display
  • Easy to scan many entries

Cons

  • Logs compete with table for space
  • Can't watch logs and scan table simultaneously

B: Split Panel (master-detail)

Left: scrollable entry list with status. Right: live log stream for the selected entry. Both panels update in real-time. Click an entry to watch its logs. Shows run number to distinguish re-runs.

Pros

  • Watch logs while scanning entries
  • Natural for monitoring workflows
  • Run isolation is visible (Run #1, #2)

Cons

  • Less column data visible per entry
  • Needs enough screen width

C: Card Grid + Slide-out Drawer

Each entry is a card with status, current step, and progress bar. Click a card to open a full-height slide-out drawer with the log stream. More visual and glanceable than a table.

Pros

  • Most visual — status at a glance
  • Progress bars show completion
  • Cards feel modern and polished

Cons

  • Less data density than table
  • Drawer covers content when open
  • Scales poorly with 50+ entries
